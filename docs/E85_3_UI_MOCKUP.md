## Color Scheme (Dark Theme)

### Background Colors
- Component background: `bg-gray-900` (#111827)
- Section background: `bg-gray-800/30` (#1F2937 with opacity)
- Card background: `bg-gray-800` (#1F2937)
- Hover background: `bg-gray-700` (#374151)

### Border Colors
- Main border: `border-gray-800` (#1F2937)
- Section border: `border-gray-700` (#374151)

### Text Colors
- Primary heading: `text-purple-400` (#C084FC)
- Secondary text: `text-gray-300` (#D1D5DB)
- Muted text: `text-gray-400` (#9CA3AF)
- Success text: `text-green-300` (#86EFAC)
- Error text: `text-red-300` (#FCA5A5)

### Status Colors
- Enabled state: `bg-green-900/10` with `border-green-700`
- Blocked state: `bg-gray-800/30` with `border-gray-700`
- Blocker section: `bg-orange-900/10`

### Button Colors
- Primary action: `bg-green-600 hover:bg-green-700`
- Disabled: `opacity-60`

## Accessibility Features

1. **Semantic HTML**
   - Proper heading hierarchy (h3, h4)
   - Form controls with labels
   - Radio buttons for state selection

2. **Visual Indicators**
   - Icons for quick recognition
   - Color + text (not color-alone)
   - Clear enabled/disabled states

3. **Keyboard Navigation**
   - Tab through radio buttons
   - Space to select
   - Enter to submit transition

4. **Screen Reader Support**
   - Descriptive labels
   - ARIA attributes for state
   - Clear button text

---

## Responsive Behavior

### Desktop (> 1024px)
- Full width component
- Multi-column layout for next states
- Larger fonts and padding

### Tablet (768px - 1024px)
- Single column layout
- Adjusted padding
- Maintained readability

### Mobile (< 768px)
- Stacked layout
- Touch-friendly buttons
- Reduced padding
- Scrollable if needed

---

## Interactive Behaviors

### On Load
1. Fetch state flow data from API
2. Display loading spinner
3. Render component when data arrives
4. Show error if fetch fails

### User Interaction
1. Click radio button to select next state
2. Button appears when valid state selected
3. Click button to request transition
4. Callback fired with new state
5. Success/error message shown

### Real-time Updates
- Component refetches on issue ID change
- Can be refreshed manually
- Updates on state change
- Optimistic UI updates possible
