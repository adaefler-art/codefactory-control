## User Interactions

1. **Page Load**: Automatically fetches migration parity data
2. **Refresh Button**: Manually triggers new parity check
3. **Limit Selector**: Changes result limit and refetches data
4. **Auto-refresh**: On limit change (via useEffect dependency)

## Design Principles

1. **Clarity**: Status immediately visible with color-coded badge
2. **Context**: Database and repo stats side-by-side for comparison
3. **Actionability**: Clear indication of what needs fixing (missing, extra, mismatches)
4. **Evidence**: Specific filenames listed for debugging
5. **Simplicity**: Clean, minimal design focused on essential information
6. **Responsiveness**: Works on all screen sizes
7. **Progressive Disclosure**: Success state simple, failure state detailed

## Screenshot Notes

Since this is a server-side rendered page requiring authentication, actual screenshots would show:

1. **Success State**: Green PASS badge with "All Migrations in Sync" message
2. **Failure State**: Red FAIL badge with expandable sections showing discrepancies
3. **Loading State**: Centered loading message during data fetch
4. **Error State**: Red error banner with actionable error message

The UI follows the AFU-9 Control Center design system with:
- Tailwind CSS utility classes
- Consistent spacing (p-6, mb-6, gap-4)
- Shadow and rounded corners on cards
- Responsive grid layouts
- Professional color palette (grays, blues, greens, yellows, reds)
